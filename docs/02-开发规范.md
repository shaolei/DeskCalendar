# 02-开发规范

> 版本：v1.0-draft ｜ 最后更新：2026-07-07

## 1. 语言与工具链

- **Go 1.25+**（ADR-06）。CI 与本地必须 ≥ 1.25。
- **零 CGO**：构建始终 `CGO_ENABLED=0`。任何引入 `import "C"` 的 PR 直接拒绝。
- 格式化：`gofmt` + `goimports`（CI 强制）。
- Lint：`golangci-lint`（启用 errcheck / staticcheck / ineffassign / revive）。
- 模块路径：`github.com/shaolei/DeskCalendar`。

## 2. 代码风格

- 包名小写单词（`shell` / `platform` / `state`），不含下划线。
- 公开符号用 `PascalCase`；内部用 `camelCase` 或 `lowerCamel`。
- 错误：函数返回 `(T, error)`；用 `fmt.Errorf("...: %w", err)` 包装；禁止裸 `panic`（仅 `main` 初始化失败可 `log.Fatal`）。
- 上下文：跨 IO / 长操作的函数首参为 `context.Context`。
- 日志：统一 `internal/infra/log`（封装 `log/slog`），区分 `debug/info/warn/error`；不向 stdout 喷调试信息（托盘程序无控制台）。

## 3. 并发规范

- **窗口 / Win32 操作只在主线程**（`desktop.Run` 所在线程，已 `runtime.LockOSThread`）。
- 跨线程通信用 **channel**，禁止在 systray goroutine 直接调用窗口 API（见 `01-总体架构.md` §3）。
- 共享状态走 `state` 包的 Signal / Store，不在业务代码里手写 `sync.Mutex` 管理 UI 状态。
- 用 `select { case ch <- v: default: }` + `RequestRedraw()` 避免阻塞发送方。

## 4. 测试

- 单元测试：`*_test.go`，覆盖率目标 ≥ 60%（核心 domain ≥ 80%）。
- **可测性**：Service / Repository / Provider 全接口化，便于 `gomock` / 手写 fake。
- 纯逻辑（农历换算、节假日判定、布局计算）必须单测，不依赖 GPU / 窗口。
- UI 集成测试允许弱：用 `testing` + 无头断言，不强求像素级。
- Spike / POC 放 `poc/`，不进 `internal/`。

## 5. 依赖管理

- 仅纯 Go 依赖（零 CGO）。新增依赖前在 PR 说明零 CGO 依据。
- `go.mod` 用 `replace` 指向本地 gogpu fork 仅在开发期；发布前切回 tagged 版本。
- 数据类依赖（holiday-cn）以 vendored JSON 嵌入，不运行时强依赖网络。

## 6. 提交与 PR

- 分支：`main` 受保护；功能分支 `feat/<scope>`、修复 `fix/<scope>`、文档 `docs/<scope>`。
- Commit：`<type>(<scope>): <subject>`，type ∈ {feat, fix, docs, refactor, test, build, chore}。
- PR：必须关联 Issue / ADR；描述含"变更 / 权衡 / 验证方式"。
- 每个 ADR 级决策变更需同步更新 `docs/` 对应文档与评估报告 ADR 表。

## 7. 文档规范

- 所有模块文档遵循 `_模板与写作规范.md` 的 **10 节结构**。
- 图用 `mermaid`（classDiagram / flowchart / sequenceDiagram）。
- UI 原型用 ASCII 框图。
- 接口用真实 Go 签名（可编译风格）。
- 中文撰写，术语中英并存（Signal / Provider / Store 等保留英文）。

## 8. 安全与隐私

- 天气为唯一联网源；默认 Open-Meteo（免 key），用户可选填和风 key。
- 不收集任何用户数据；不内置遥测。
- 开机自启通过注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`，仅当前用户。
