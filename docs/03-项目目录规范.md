## 1. 文档目录（`docs/`）

文档编号前缀仅用于**阅读顺序与归类**，不代表 Go 包名。

```
docs/
├── 00-项目介绍.md
├── 01-总体架构.md
├── 02-开发规范.md
├── 03-项目目录规范.md   ← 本文
├── _模板与写作规范.md
├── 10-Shell/      App Window Layout Lifecycle
├── 20-Platform/   WorkerW WindowStyle Mica Acrylic DPI MultiMonitor Startup Tray Notification
├── 30-State/      Signal Store DataFlow
├── 40-Theme/      Theme ThemeJson Icon Font Skin
├── 50-Calendar/   Calendar Month Week Lunar Holiday
├── 60-Todo/       Model SQLite Reminder
├── 70-Weather/    Provider Cache API
├── 80-Plugin/     Plugin Lifecycle Event
├── 90-UI/         MainWindow CalendarView TodoView WeatherView Settings Animation
└── 100-Release/   Build CI Package AutoUpdate
```

命名规则：

- 目录：`两位数字-模块名`（数字为 tens 分组，便于排序）。
- 文件：每个子主题一个 `.md`，文件名即主题（如 `App.md` / `Lunar.md`）。
- 现有交付物保留：`DeskCalendar_完整架构设计文档.md`、`DeskCalendar_技术评估报告_gogpu-ui-Windows.md`、`ADR-05-数据源选型对比分析.md`。

## 2. 代码目录（`internal/` 与 `pkg/`）

文档模块 → Go 包映射：

```
deskcalendar/
├── main.go                  # 仅做 wire（组装 app），不含业务
├── build/                   # 仓库根（非 internal/）：构建期纯资源包（NSIS 脚本/目标产物/资源），零运行时逻辑
│   ├── packaging/           # 打包脚本（NSIS / Inno）
│   ├── nsis/                # NSIS 插件与脚本
│   ├── target/              # 构建目标定义
│   └── assets/              # 构建期资源
├── internal/
│   ├── app/                 # 应用编排（desktop.Run 装配）
│   ├── shell/               # 10-Shell：Lifecycle / Window / Layout
│   ├── platform/            # 20-Platform：tray / windowstyle / dpi / multimonitor / startup
│   ├── state/               # 30-State：signal / store / dataflow / event（事件总线实现）
│   ├── theme/               # 40-Theme：theme / themejson / icon / font / skin
│   ├── calendar/            # 50-Calendar：calendar / month / week / lunar / holiday
│   ├── todo/                # 60-Todo：model / sqlite / reminder
│   ├── weather/             # 70-Weather：provider / cache / api
│   ├── plugin/              # 80-Plugin：plugin / lifecycle（EventBus 实现位于 state）
│   ├── ui/                  # 90-UI：mainwindow / calendarview / ... / settings / animation
│   ├── updater/             # 100-Release：运行时自动更新器（internal/updater）
│   └── infra/               # 跨切面：log / config / fs
└── pkg/                     # 可选：当前未使用（lunar 适配封装按 ADR-05a 落在 internal/calendar，无需 pkg/）
```

约定：

- `internal/` 不对外暴露；`pkg/` 仅放稳定、无内部依赖的纯工具。
- 依赖方向自上而下：`app → shell → {platform, state} → feature → theme`，禁止反向。
- 每个包一个 `doc.go` 描述包职责（对应模块文档的 📦 package 设计）。

```
└── data/
    └── holidays/            # holiday-cn 烘焙的 JSON（每年构建期刷新一次）
poc/                         # 临时验证，不入 internal
├── transparent-window/
└── systray-spike/
```

## 4. 配置

- 用户配置：`%AppData%/DeskCalendar/config.json`（开机自启、主题、弹窗位置、天气 key）。
- 不写注册表敏感信息；天气 key 仅存本地用户配置。
